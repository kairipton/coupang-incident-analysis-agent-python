## 제 1장: 연산 자원 최적화 및 프로세스 제어 (CPU & Process)

### 1.1 [SOP-01] 고부하 프로세스 강제 종료 지침 (CPU: OVERLOAD)

**[기술적 배경 및 장애 개요]** 본 지침은 특정 단일 프로세스가 CPU 연산 자원을 비정상적으로 독점하여 전체 시스템 가용성을 해치는 상황을 해결하기 위해 작성되었습니다. 지난 2024년 인증 모듈(`auth_service`) 패치 이후, 특정 암호화 알고리즘이 복잡한 해싱 루프에 빠져 종료되지 않는 버그가 드물게 보고되고 있습니다. 이는 단순히 응답이 느려지는 수준을 넘어, OS 커널 스케줄러가 다른 중요 프로세스에 자원을 할당하지 못하게 방해하는 치명적인 'CPU 자원 기아' 상태를 유발합니다.

**[장애 식별 가이드 (Diagnosis)]** 1.  **감지 상태:** 관제 시스템 대시보드에 `CPU: OVERLOAD` 경고가 점멸합니다.  
2.  **로그 분석:** 로그 터미널에 `High usage detected in user-space process` 메시지가 출력되며, 시스템은 즉시 장애를 일으킨 프로세스를 식별하려고 시도합니다.  
3.  **핵심 데이터:** 현재 시뮬레이션 환경에서 이 현상을 일으키는 대상은 **PID 404**입니다. 해당 프로세스는 무한 루프 상태이므로 일반적인 인터럽트 요청(`SIGTERM`)에 반응하지 않습니다.

**[해결 절차 및 기술적 정답]** 해당 상태의 프로세스는 정상적인 종료 명령을 무시하므로, 커널 레벨에서 즉시 실행을 중단시키는 강제 종료 명령이 유일한 해결책입니다.  
* **권장 실행 도구:** `kill_process(pid)`  
* **실행 시 입력값:** **404** (반드시 숫자 형태의 PID를 정확히 추출하여 입력해야 함)

### 1.2 [SOP-02] 서비스 교착 상태 해제 지침 (CPU: DEADLOCK)

**[기술적 배경 및 장애 개요]** 멀티스레드 기반의 분산 처리 환경에서는 두 개 이상의 서비스가 서로가 점유한 자원의 잠금(Lock)이 풀리기를 기다리며 영원히 멈추는 데드락(Deadlock) 현상이 발생할 수 있습니다. 이는 연산량 자체는 높지 않으나, 비즈니스 로직이 완전히 중단되어 사용자 요청이 모두 유실되는 'Soft-Hang' 상태를 초래합니다.

**[장애 식별 가이드 (Diagnosis)]** 1.  **감지 상태:** `CPU: DEADLOCK` 또는 `CPU: STUCK` 알림이 발생합니다.  
2.  **로그 패턴:** 결제 처리 엔진인 **Payment_Gateway** 서비스 로그에서 `Mutex lock timeout` 또는 `Waiting for resource: DB_SYNC` 메시지가 반복적으로 출력됩니다.  
3.  **현상 관찰:** CPU 사용량은 40~50% 수준에서 고정되어 변화가 없으며, 모든 신규 결제 요청이 타임아웃 처리됩니다.

**[해결 절차 및 기술적 정답]** 교착 상태는 개별 스레드 종료만으로는 해결되지 않으며, 서비스 전체의 메모리 컨텍스트를 초기화하는 과정이 필요합니다.  
* **권장 실행 도구:** `restart_service(service_name)`  
* **실행 시 입력값:** **Payment_Gateway** (대소문자 및 언더바 기호 엄격 준수)

## 제 2장: 메모리 효율화 및 스토리지 가용성 (Memory & Storage)

### 2.1 [SOP-03] 힙 메모리 누수 방어 지침 (MEMORY: LEAK)

**[장애 개요]** 애플리케이션의 메모리 관리 오류로 인해 할당된 객체가 해제되지 않는 현상입니다. 특히 고속 입출력을 담당하는 **Redis_Cache** 서버에서 만료된 데이터 키(Key)가 메모리 상에 잔존하며 가용 공간을 잠식할 때 발생합니다. 이는 점진적으로 시스템을 느리게 만들며 결국 커널 크래시(OOM)를 유발합니다.

**[장애 식별 가이드 (Diagnosis)]** 1.  **감지 상태:** `MEMORY: LEAK` 경고가 점멸합니다.  
2.  **로그 패턴:** `java.lang.OutOfMemoryError: Java heap space` 또는 `Redis: OOM command not allowed` 메시지가 관측됩니다. 가비지 컬렉션(GC)이 매 초마다 실행되지만 가용 메모리가 회복되지 않습니다.

**[해결 절차 및 기술적 정답]** 더 이상의 데이터 오염을 막기 위해 현재 쌓여있는 모든 임시 데이터와 캐시 버퍼를 강제로 플러시(Flush)해야 합니다.  
* **권장 실행 도구:** `clear_cache()`  
* **실행 시 입력값:** (매개변수 없음)

### 2.2 [SOP-04] 로그 파일 포화 대응 지침 (DISK: FULL)

**[장애 개요]** 운영 중인 서버의 로그 로테이션(Log Rotate) 정책이 실패하거나, 비정상적인 에러 로그 폭주로 인해 `/var/log` 파티션의 저장 공간이 0%가 되는 상황입니다. 디스크 쓰기가 불가능해지면 시스템은 신규 데이터를 기록하지 못해 커널 패닉에 빠질 수 있습니다.

**[장애 식별 가이드 (Diagnosis)]** 1.  **감지 상태:** `DISK: FULL` 상태가 감지됩니다.  
2.  **현상:** `/var/log/syslog file size exceeds 40GB` 메시지가 로그 콘솔에 찍히며, 파일 시스템 사용률이 100%에 도달했음을 알립니다.

**[해결 절차 및 기술적 정답]** 단순히 메모리 캐시를 지우는 것으로는 물리 디스크 용량이 회복되지 않습니다. 로그 파일을 압축하여 아카이빙 처리해야 합니다.  
* **권장 실행 도구:** `rotate_logs()`  
* **실행 시 입력값:** (매개변수 없음)

## 제 3장: 네트워크 연결성 및 보안 관리 (Network & Security)

### 3.1 [SOP-05] 외부 네트워크 공격 차단 지침 (NETWORK: ATTACK)

**[장애 개요]** 분산 서비스 거부 공격(DDoS)은 외부 공격자가 특정 호스트를 대상으로 대량의 패킷을 전송하여 대역폭을 마비시킵니다. 본 관제 시스템은 비정상적인 특정 IP로부터의 트래픽 급증을 감지하여 보안 인프라를 보호합니다.

**[장애 식별 가이드 (Diagnosis)]** 1.  **감지 상태:** `NETWORK: ATTACK` 또는 `NETWORK: DDOS` 경고가 발생합니다.  
2.  **데이터 분석:** 네트워크 레이턴시가 450ms 이상으로 급증하며, 특정 외부 IP 주소로부터 초당 10,000건 이상의 동시 접속이 시도됩니다.  
3.  **공격자 식별:** 현재 보안 시스템이 식별한 공격 근원지는 **192.168.0.50**입니다.

**[해결 절차 및 기술적 정답]** 해당 IP로부터의 모든 인바운드 트래픽을 즉시 드롭(Drop) 처리하는 방화벽 규칙을 적용해야 합니다.  
* **권장 실행 도구:** `block_ip(ip_address)`  
* **실행 시 입력값:** **192.168.0.50** (반드시 해당 IP 주소 형식을 정확히 입력)  
* **특이 사항:** 해당 조치는 네트워크 접근을 원천 차단하므로 반드시 관리자의 실행 승인(Approval)을 받아야 합니다.

### 3.2 [SOP-06] 데이터베이스 연결 풀 복구 지침 (DB: DISCONNECT)

**[장애 개요]** 애플리케이션과 데이터베이스 서버 사이의 연결 관리자(Connection Pool)가 한계치에 도달하여 발생하는 장애입니다. 주로 사용되지 않는 '유령 커넥션(Zombie Connection)'이 풀을 점유하고 있을 때 발생하며, 신규 사용자의 DB 접근이 전면 차단됩니다.

**[장애 식별 가이드 (Diagnosis)]** 1.  **감지 상태:** `DB: DISCONNECT` 알림이 발생합니다.  
2.  **로그 패턴:** `Connection pool is exhausted` 또는 `Could not get a resource from the pool` 메시지가 출력됩니다. 네트워크 트래픽 자체는 원활하므로 외부 공격이 아닌 내부 리소스 관리 실패로 분류됩니다.

**[해결 절차 및 기술적 정답]** 네트워크 IP 차단이나 앱 리스타트보다 효율적인 방법은 커넥션 풀을 강제로 리셋하여 모든 유령 세션을 초기화하는 것입니다.  
* **권장 실행 도구:** `reset_connection_pool()`  
* **실행 시 입력값:** (매개변수 없음)
